# DBTT Car Modification — Vos Automotive Digital Platform

A digital customization platform for **Vos Automotive (Singapore)** that lets customers visualise car modifications in 3D, book appointments, and provides owners and workers with a live analytics dashboard.

The project is split into two parts that must both be running at the same time:

| Part | Technology | Port |
|---|---|---|
| **Frontend** | React + Vite + TypeScript | `http://localhost:5173` |
| **Backend** | Python + FastAPI | `http://localhost:8000` |

---

## Setup & Running

You need **two separate terminals** open at the same time — one for the backend, one for the frontend.

---

### Terminal 1 — Backend (Python / FastAPI)

```bash
# 1. Navigate to the backend folder
cd backend

# 2. (Recommended) Create and activate a virtual environment
python -m venv venv

# On Windows (Git Bash) — use this:
source venv/Scripts/activate

# On Windows (Command Prompt):
# venv\Scripts\activate.bat

# On Windows (PowerShell):
# venv\Scripts\Activate.ps1

# On macOS/Linux:
# source venv/bin/activate

# 3. Install Python dependencies
pip install -r requirements.txt

# 4. Start the backend server
python -m uvicorn main:app --reload
```

> **Note:** Use `python -m uvicorn` instead of just `uvicorn` — this avoids PATH issues on Windows where the command may not be recognised even after installing.

The backend is ready when you see:
```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
```

You can verify it is working by opening `http://localhost:8000/api/kpis` in your browser — it should return JSON.

---

### Terminal 2 — Frontend (React / Vite)

```bash
# 1. Navigate to the frontend folder
cd frontend

# 2. Install Node dependencies (only needed once, or after pulling changes)
npm install

# 3. Start the development server
npm run dev
```

The frontend is ready when you see:
```
  VITE v6.x.x  ready in Xms

  ➜  Local:   http://localhost:5173/
```

Open `http://localhost:5173` in your browser.

## API Endpoints (Backend)

All endpoints are read-only (`GET`). The backend runs on `http://localhost:8000`. All data is computed from `analysis.py`.

| Endpoint | Description |
|---|---|
| `GET /api/projections` | Historical data (2020–2024) + baseline vs digital revenue forecast (2025–2029) |
| `GET /api/service-stats` | Popular services, top bundles, AOV by service, budget bands, style preferences |
| `GET /api/recommendations` | Cross-sell recommendation rules with confidence scores |
| `GET /api/uplift-kpis` | Headline digital transformation KPIs (5-year uplift summary) |
| `GET /api/inventory-predictions` | Predictive stock alerts with days-until-stockout and reorder quantities |
| `GET /api/kpis` | Four operational KPI cards (revenue, orders, AOV, top service) |
| `GET /api/monthly-revenue` | 2025 annual revenue distributed across 12 months using seasonal weights |
| `GET /api/service-breakdown` | Revenue contribution per service (primary count × AOV) |
| `GET /api/trend-insights` | H1 vs H2 2024 service demand comparison derived from transaction dates |
| `GET /api/service-bundles` | Top service bundles (used by the worker dashboard) |

Interactive API docs (auto-generated by FastAPI): `http://localhost:8000/docs`

## Backend — How It Works

The backend contains two Python files.

### `main.py` — API Entry Point

`main.py` is the FastAPI application that the server runs. It does three things:

1. Creates the FastAPI app and configures CORS so the React frontend (port 5173) is allowed to call it.
2. Imports functions from `analysis.py`.
3. Registers each function as a GET endpoint (e.g. `GET /api/projections` calls `get_projections()` from `analysis.py`).

Nothing is computed inside `main.py` itself — it is purely the routing layer.

---

### `analysis.py` — All Analytics as Python Functions

`analysis.py` is a port of the business analysis notebook (`analysis.ipynb`) into callable Python functions. Every number shown on the owner dashboard originates here.

All data is synthetic and generated deterministically from fixed assumptions (defined in `_ASSUMPTIONS`) and a seeded random number generator (`np.random.default_rng(42)`), so results are identical on every run.

#### Functions

| Function | Type | What it produces |
|---|---|---|
| `get_projections()` | Predictive | Synthetic historical data (2020–2024) plus a 5-year forecast (2025–2029) under two scenarios: **Baseline** (organic growth only) and **Digital** (growth plus conversion and upsell lifts). Returns both scenarios, a year-by-year comparison, and the `baselineGrowthRate` used. |
| `get_service_stats()` | Descriptive | Generates 320 synthetic customer transactions then computes: most popular services, top co-purchased bundles, average order value per service, customer budget band distribution, and style preference breakdown. |
| `get_recommendations()` | Prescriptive | Association rule mining on the 320-transaction dataset. For each service, finds the service most frequently co-purchased and returns the confidence score — powers the cross-sell recommendation table. |
| `get_uplift_kpis()` | Prescriptive | Aggregates projection and analytics outputs into headline KPIs: 5-year customer/revenue uplift, average annual uplift, top bundle, top service, and strongest cross-sell rule. |
| `get_inventory_predictions()` | Predictive + Prescriptive | Derives weekly demand per inventory item from transaction frequency, computes days-until-stockout, classifies urgency (critical ≤7d / warning ≤14d / healthy), and recommends reorder quantities. |
| `get_kpi_stats()` | Descriptive | Four dashboard KPI cards: total revenue from 2025 digital projection, orders/month from projected customer count, mean AOV from transaction data, top service by mention count. |
| `get_monthly_revenue()` | Predictive | Distributes the 2025 digital projection annual revenue across 12 months using a seasonal weighting pattern. Final month absorbs any rounding remainder. |
| `get_service_breakdown()` | Descriptive | Revenue contribution per service computed as `primary_customer_count × avg_order_value` from the transaction dataset. |
| `get_trend_insights()` | Descriptive | Compares H1 (Jan–Jun) vs H2 (Jul–Dec) 2024 primary service demand using synthetic transaction dates. Returns the top 6 services by magnitude of change. |

## Navigating the App

With both servers running, open `http://localhost:5173` in your browser. All sections are accessible from the landing page.

### Customer flow

1. **Design & Book** — click **Design & Book** on the hero (or **Customize Your Car** in the nav) to open the 3D customizer
   - Pick a body colour, finish, window tint, rim colour, caliper colour, and optional mods using the panel on the right
   - When happy with the design, scroll to the bottom of the panel and click **Book Appointment** — your selections are carried over automatically
   - Complete the 5-step booking form (services → date & time → logistics → your details → confirm)
   - On the confirmation screen your **booking reference** (e.g. `APT-006`) is shown — keep this to track your booking

2. **Track a booking** — click **Track Booking** in the nav or **Track My Booking** on the hero
   - Enter your booking reference (e.g. `APT-001`) or the email address used when booking
   - The page shows your current status, a 4-step progress timeline, estimated duration, and logistics details
   - When your car is ready, a **WhatsApp** button appears to arrange collection directly with the workshop

> **Demo bookings to try:** `APT-001` through `APT-005` are pre-loaded. APT-003 is in-progress, APT-004 is completed.

---

### Worker flow

1. Click **Staff Access** in the nav (or footer) → select **Worker**
2. The worker dashboard shows all appointments in a 3-column Kanban board: **New Bookings → In Progress → Completed**
3. Click **Start Job** on a card to move it to In Progress — this immediately updates the customer's tracking page
4. Click **Mark Done** to move it to Completed — the customer's tracking page shows the WhatsApp collection button

---

### Owner flow

1. Click **Staff Access** in the nav (or footer) → select **Owner**
2. The owner dashboard loads analytics from the backend — make sure the backend server is running on port 8000
3. Panels include: live bookings KPIs, revenue projection, monthly revenue, service breakdown, AOV by service, customer budget segments, cross-sell recommendations, trend insights, and predictive supply alerts

## Optional: AI Chatbot (Customizer)

The AI chatbot on the customizer page requires an OpenAI API key. Without it the chatbot will not respond, but the rest of the app works normally.

Create a `.env` file inside the `frontend/` folder:

```
VITE_OPENAI_API_KEY=sk-...your-key-here...
```
